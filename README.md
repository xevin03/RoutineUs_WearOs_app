# ---------------------------

# 1) 워치 개발자 옵션 활성화

# ---------------------------

# - 워치: 설정 → 워치 정보 → 소프트웨어 정보

# - '소프트웨어 버전(빌드 번호)' 7번 탭 → 개발자 옵션 켜짐

# - 설정 → 개발자 옵션 들어가서:

# \* ADB 디버깅: ON

# \* Wi-Fi 디버깅(무선 디버깅): ON

# - 켜면 워치 화면에 "IP주소:포트" 와 "페어링 코드"가 표시됨

# ---------------------------

# 2) PC에서 ADB 서버 실행

# ---------------------------

adb kill-server # ADB 서버 리셋
adb start-server # 새로 시작

# ---------------------------

# 3) 워치와 ADB 페어링

# ---------------------------

# (A) 페어링 단계: 워치에 뜬 '페어링 포트'로 진행

adb pair <워치IP:페어링포트>

# 예시: adb pair 192.168.0.23:41075

# → 실행하면 "페어링 코드" 입력 요청이 뜨는데, 워치에 표시된 코드 입력

# (B) 연결 단계: 워치의 '디버그 포트'로 connect

adb connect <워치IP:디버그포트>

# 예시: adb connect 192.168.0.23:5555

# ---------------------------

# 4) 연결 확인

# ---------------------------

adb devices

# → 리스트에 "192.168.0.23:5555 device" 이런 식으로 떠야 정상 연결

# ---------------------------

# 5) Flutter에서 워치 실행

# ---------------------------

flutter devices

# → 워치가 디바이스 목록에 표시됨 (IP:포트 형태)

flutter run -d <워치\_디바이스ID>

# 예시: flutter run -d 192.168.0.23:5555

# ---------------------------

# 6) APK 직접 설치 (선택)

# ---------------------------

flutter build apk --release
adb install -r build/app/outputs/flutter-apk/app-release.apk

# ---------------------------

# ⚠️ 자주 나는 문제 해결

# ---------------------------

# - pair 성공했는데 connect 실패 → "페어링 포트"랑 "디버그 포트" 구분 필요

# - device 가 'offline' 상태 → 워치 화면 깨우고 다시 시도

# - 두 개 디바이스 잡힐 때 → flutter run -d <정확한ID> 지정

# - 갤럭시 워치3 이하(Tizen) → ADB/Flutter 아님, Tizen Studio 필요
